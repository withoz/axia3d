## 🧪 진단 테스트 방법

### Step 1: 간단한 DXF 파일로 테스트

**테스트 파일:** `simple_test.dxf`
- 위치: `/web/dist/simple_test.dxf`
- 내용: 간단한 선 2개 + 원 1개
- 크기: 334 bytes (매우 작음)

**테스트 절차:**
```
1. AXiA 3D 콘솔 클리어 (F12 → Console)
2. File → Import → DXF (.dxf) 클릭
3. simple_test.dxf 파일 선택
4. 콘솔 메시지 확인
```

### Step 2: 콘솔 메시지 해석

**성공 시 보여야 할 메시지:**
```
[FileImporter] DXF 가져오기: simple_test.dxf
[FileImporter] 완료: simple_test.dxf — 1 메시, ...
```

**실패 시 보여질 메시지:**
```
❌ Error: [구체적인 에러 메시지]
❌ TypeError: ...
❌ Uncaught: ...
```

### Step 3: 구체적인 에러 수집

콘솔에서 **첫 번째 빨간 에러 메시지**를 완전히 복사해주세요.

## 🔍 예상 가능한 문제들

### 가능 1: DXF 파서 라이브러리 오류
```
❌ Error: Cannot read property 'parse' of undefined
```
**원인:** DXF 파서 라이브러리가 제대로 로드되지 않음

### 가능 2: 파일 형식 오류
```
❌ SyntaxError: Unexpected token...
```
**원인:** DXF 파일 형식이 잘못됨

### 가능 3: 엔티티 변환 오류
```
⚠️ DXF에서 렌더링 가능한 엔티티를 찾을 수 없습니다
```
**원인:** DXF 파일에 지원되는 엔티티가 없음

### 가능 4: DWG 변환 오류
```
❌ Error: convertDwgToDxf is not a function
```
**원인:** dwgdxf 라이브러리가 제대로 초기화되지 않음

## 📊 DXF/DWG 구현 상태

### DXF 지원 엔티티
- ✅ LINE (선)
- ✅ CIRCLE (원)
- ✅ ARC (호)
- ✅ LWPOLYLINE / POLYLINE (폴리선)
- ✅ SOLID / FACE (면)
- ✅ 3DFACE (3D면)

### DWG 처리 흐름
```
DWG 파일
    ↓
[dwgdxf] DWG → DXF 변환
    ↓
[DXF 파서] DXF 파싱
    ↓
[엔티티 변환] 3D 메시 생성
    ↓
[libredwg-web] 메타데이터 추출 (선택사항)
    ↓
Three.js Group 반환
```

## 🔧 문제 해결 방법

### A) 에러 메시지 없고 아무것도 안 됨
**가능한 원인:**
- 파일 선택 대화가 안 열림 → 메뉴 이벤트 미처리
- **해결:** 이미 메뉴 이벤트는 고정됨 (SKP 테스트로 확인)

### B) "파서 에러" 메시지 보임
**가능한 원인:**
- DXF 파일 형식이 잘못됨
- **해결:** simple_test.dxf로 시작 (검증된 파일)

### C) "엔티티를 찾을 수 없다" 메시지 보임
**가능한 원인:**
- DXF 파일에 지원되는 엔티티가 없음
- **해결:** 다른 DXF 파일 시도 또는 엔티티 타입 확장

### D) "라이브러리 로드 오류" 메시지 보임
**가능한 원인:**
- dxf 또는 dwgdxf npm 패키지 문제
- **해결:**
  ```bash
  cd "AXiA 3D/web"
  npm install
  npm run build
  ```
