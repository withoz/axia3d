================================================================================
TECHNICAL HIGHLIGHTS
================================================================================

CLEAN INTERFACE PATTERN:
  - Each tool implements ITool interface
  - All tools receive ToolContext with shared state
  - No direct access to ToolManager
  - Eliminates tight coupling

TYPE SAFETY:
  - Full TypeScript strict mode
  - No 'any' types (except where necessary for DOM/THREE.js)
  - Proper generics and discriminated unions

EVENT DISPATCH:
  - Simple map-based tool lookup: O(1)
  - Mouse events delegated to active tool
  - Keyboard events for axis lock and escape

SHARED STATE MANAGEMENT:
  - faceMap, edgeMap, units, snap, selection all shared
  - Tools don't duplicate state
  - syncMesh() coordinates all updates

HELPER METHODS (ToolManager → Tools):
  - getSnappedPoint() — snap calculation
  - getGroundPoint() — plane intersection
  - get3DPoint() — mesh or plane hit
  - extractFaceBoundary() — face outline for ghosts
  - getAxisInferredPoint() — axis detection
  - updateAxisGuide() — axis visualization

================================================================================
EXTENSIBILITY
================================================================================

ADDING A NEW TOOL (4 steps):

1. Create MyNewTool.ts:
   export class MyNewTool implements ITool {
     readonly name = 'mytool';
     constructor(ctx: ToolContext) { this.ctx = ctx; }
     onMouseDown(e, point) { /* ... */ }
     onMouseMove(e, point) { /* ... */ }
     isBusy() { return /* ... */; }
     // ... other methods
   }

2. Register in ToolManagerRefactored:
   this.tools.set('mytool', new MyNewTool(this.toolContext));

3. Update UI buttons to call:
   toolMgr.setTool('mytool');

4. That's it! No modification to event handlers needed.

MAINTENANCE NOTES

ADDING A NEW FEATURE:
  1. Identify which tool needs the feature
  2. Modify that tool's implementation
  3. No changes to ToolManager or other tools needed
  4. Benefit: Changes are localized and testable

FIXING A BUG:
  1. Find the tool with the bug
  2. Fix only that tool
  3. No risk of affecting other tools
  4. Benefit: Isolated testing of fix

REMOVING A TOOL:
  1. Delete the tool file
  2. Remove from tools.set() in ToolManager
  3. Remove from main.ts button mapping
  4. Done! No other changes needed
