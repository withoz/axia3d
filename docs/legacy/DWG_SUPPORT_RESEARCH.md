# DWG 속성 추출 및 지원 방안 검토

**날짜:** 2026-04-13  
**목표:** DWG 파일의 속성(메타데이터, 레이어, 엔티티 정보)을 추출 가능한 방법 평가

---

## 검토된 솔루션

### 1️⃣ **@mlightcad/libredwg-web** ⭐⭐⭐⭐⭐

**개요:** WebAssembly 기반 LibreDWG (GNU)  
**npm:** `@mlightcad/libredwg-web`

#### 장점
- ✅ DWG 완전 파싱 (R14 ~ 2020)
- ✅ 메타데이터 추출 (제목, 작성자, 단위, 버전 등)
- ✅ 레이어 정보 추출
- ✅ 엔티티 정보 및 속성 추출
- ✅ DXF 변환 가능
- ✅ 브라우저 네이티브 지원
- ✅ TypeScript 지원
- ✅ 활발한 유지보수

#### 단점
- ⚠️ WASM 파일 크기 (~2-3MB)
- ⚠️ 파싱 성능 (대용량 파일)

#### 추정 구현 시간
- **메타데이터만:** 2-3시간
- **전체 엔티티:** 1-2일

---

### 2️⃣ **dwgdxf** ⭐⭐⭐⭐

**개요:** DWG → DXF 변환 (ACadSharp WASM)  
**npm:** `dwgdxf`

#### 장점
- ✅ DWG를 DXF로 변환 (우리는 이미 DXF 지원!)
- ✅ 간단한 구현
- ✅ 기존 DXF 파이프라인 활용 가능
- ✅ 메타데이터 보존

#### 단점
- ⚠️ DWG 속성을 직접 추출하지는 않음 (DXF로 변환하므로 간접)
- ⚠️ 변환 프로세스 필요

#### 추정 구현 시간
- **1-2시간** (매우 간단)

#### 구현 전략
```typescript
// 사용 흐름:
1. DWG 파일 읽기
2. dwgdxf로 DXF로 변환
3. 기존 DXF 파서로 파싱
4. 엔티티 및 메타데이터 추출
```

---

## 추천 구현 전략

### **Option A: dwgdxf 사용** (단기, 빠른 구현)

**구현 시간:** 1-2시간  
**복잡도:** 낮음  
**신뢰도:** 높음

**흐름:**
```
DWG 파일 → dwgdxf (변환) → DXF → 기존 DXF 파서 → 메타데이터 + 엔티티
```

**장점:**
- 기존 DXF 파이프라인 활용
- 구현 간단
- 검증됨

**단점:**
- DWG 직접 파싱 아님 (간접)
- 변환 성능 오버헤드

---

### **Option B: @mlightcad/libredwg-web 사용** (중기, 완벽 지원)

**구현 시간:** 1-2일  
**복잡도:** 중간  
**신뢰도:** 매우 높음

**흐름:**
```
DWG 파일 → libredwg-web (파싱) → 메타데이터 + 엔티티 직접 추출
```

**장점:**
- DWG 직접 파싱
- 메타데이터 완전 추출
- 성능 최적
- 레이어/블록 정보 추출

**단점:**
- WASM 번들 크기 증가
- 구현 복잡도 높음

---

### **Hybrid 전략** (권장) ⭐⭐⭐⭐⭐

**Phase 1 (즉시):** dwgdxf로 기본 지원
```bash
npm install dwgdxf
```

**Phase 2 (향후):** @mlightcad/libredwg-web로 완벽 지원 전환

## 추출 가능한 DWG 속성들

### 메타데이터
- 파일 버전 (AutoCAD R14 ~ 2020+)
- 도면 크기 (Width, Height)
- 단위 (mm, cm, in, ft 등)
- 제목, 주제, 작성자
- 생성/수정 일시

### 레이어 정보
- 레이어 이름
- 색상 (ACI 또는 RGB)
- 선 타입 (Continuous, Dashed 등)
- 가시성 (Visible/Hidden)
- 잠금 상태

### 엔티티 정보
- 타입 (LINE, CIRCLE, POLYLINE, INSERT 등)
- 좌표 및 크기
- 색상 및 스타일
- 속성 (레이어, 선 가중치 등)
- 블록 참조 정보

### 블록 정보
- 블록 이름
- 블록 정의 좌표
- 포함된 엔티티 목록

## 구현 로드맵

### **Week 1: dwgdxf 기본 지원**
```
목표: DWG 파일을 DXF로 변환 후 기존 파이프라인 활용
예상 시간: 2-3시간
```

**할 일:**
1. npm 패키지 설치
2. FileImporter에 DWG 로더 추가
3. dwgdxf로 DXF 변환
4. 기존 DXF 파서 활용

**결과:**
- DWG 파일 → DXF 변환 → 뷰포트 렌더링

---

### **Week 2-3: @mlightcad/libredwg-web 완벽 지원**
```
목표: DWG 직접 파싱, 메타데이터 추출 UI 추가
예상 시간: 1-2일
```

**할 일:**
1. libredwg-web 설치
2. DWG 파서 구현
3. 메타데이터 추출 로직
4. 레이어/엔티티 정보 UI 패널

**결과:**
- DWG 속성 검사 패널
- 레이어 트리 뷰
- 메타데이터 표시
