### 1.1 현재 상태 (2026-05-04 이후, ADR-051 P-2 closure)

- 큰 RECT 안에 작은 RECT × 2 *인접* (edge 공유 또는 vertex 공유)

**현재 동작** (ADR-015 fallback at scene.rs:3208):
- ADR-015 single-promote heuristic 가 connected case 에서 작동
- 두 inner 가 *별개* simple face 로 공존
- Container 는 *ring* 으로 rebuild 되지만 *combined-perimeter* 가 아닌
  *single-promote* path → 결과적으로 1 nm edge 잔존 (shared y=0 boundary)
- Face existence 보존 + 시각 렌더링 정상

**Deferred 한계** (ADR-051 §2.5):
- 진정한 ring-with-hole rebuild — connected component → 1 combined hole
- combined-perimeter 계산 + multi-loop face 재구축 경로 강화 필요
- 현재 `test_p7_canonical_stacked_inner_manifold` 가 `<=1` nm edge 로

### 2.2 ADR-051 §2.3 spec 답습 가능 (스펙 이미 있음)

`enforce_p7_canonical` 헬퍼 (ADR-051 §2.3.1) 스펙:
```rust
fn enforce_p7_canonical(container: FaceId, inners: &[FaceId]) -> Result<()> {
    // (1) Component grouping
    let components = find_inner_components(inners);
    // (2) Each component → combined perimeter (hole loop)
    let hole_loops = components.iter()
        .map(|c| compute_combined_perimeter(c).map(|p| reverse_to_cw(p)))
        .collect::<Result<Vec<_>>>()?;
    // (3) Container 를 ring face 로 재구성
    let new_container = rebuild_as_ring_face(container, &hole_loops)?;
    // (4) Inner sub-face 들은 별개 simple face 로 유지 (변경 없음)
    // (5) Invariant 검증
    debug_assert!(verify_p7_manifold(&new_container, inners).is_valid());
    Ok(())
}
```

**핵심**: 모든 step (1-5) 의 building block 이 *이미 존재*. assembly + debug_assert 만.
